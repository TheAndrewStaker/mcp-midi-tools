MCP MIDI Tools v0.1.0
=====================

A local MCP server that lets Claude Desktop talk to your USB MIDI
gear over plain English. First-class support for the Fractal AM4,
Fractal Axe-Fx II XL+, and ASM Hydrasynth Explorer. Generic MIDI
primitives work against any USB MIDI device the OS exposes.

INSTALL (3 steps)
-----------------

1. Make sure Claude Desktop is closed (system tray right-click then
   Quit, not just the window's X).

2. Double-click setup.cmd. A console window opens, registers this
   folder with Claude Desktop, and waits for you to press a key.

3. Open Claude Desktop. Start a new chat. The mcp-midi-tools server
   appears in the connector panel (the + button near the chat input).

That's it. Ask Claude something like "Using mcp-midi-tools, list the
MIDI ports you can see" to confirm your gear is reachable.

If your gear's tools don't appear, or Claude says it can't find the
device, double-click verify-midi.cmd. It asks the OS directly which
MIDI devices it sees and tells you whether your AM4 / Axe-Fx II /
Hydrasynth is reachable — useful for ruling out a driver / cable /
power issue before troubleshooting Claude Desktop.

WILL THIS WRECK MY PRESETS?
---------------------------

No, and the safety is enforced by the server, not by trust in the AI:

- The AI cannot save to a slot unless you used save-language ("save
  this", "store as M03", "keep it"). Bare "build a tone at G01"
  reads as audition, NOT save. The server refuses silent saves with a
  visible refusal message in your chat.

- If you've been editing a preset and the AI tries to load a different
  one, the server refuses with a warning naming what would be lost.
  You see the warning, you tell the AI save-or-discard.

- Z04 is the conventional scratch slot. Tell the AI to use any other
  letter if you've claimed Z04 for something personal.

- Factory restore is one tool call away ("restore A01 to factory")
  if you do somehow clobber something.

For the full safety model + what to say to the AI to get the result
you want, read SAFETY-FOR-MUSICIANS.md in the docs folder of the
GitHub repo:
https://github.com/TheAndrewStaker/mcp-midi-tools/blob/main/docs/SAFETY-FOR-MUSICIANS.md

REQUIREMENTS
------------

- Windows 10 or 11.
- At least one supported device connected by USB:
  * Fractal AM4 — driver: https://www.fractalaudio.com/am4-downloads/
  * Fractal Axe-Fx II XL+ — uses the Fractal USB driver (same family)
  * ASM Hydrasynth Explorer — class-compliant USB MIDI, no driver needed
  (Unsupported devices still reachable through the generic-MIDI tools.)
- Claude Desktop installed: https://claude.ai/download

UNINSTALL
---------

1. Double-click uninstall.cmd to remove the entry from Claude
   Desktop's config (any other MCP servers you have stay intact).
2. Delete this folder.

FAQ
---

Q: Windows says "this file came from another computer." Is it safe?
A: Yes. v0.1.0 ships unsigned to keep the project free; the source is
   open at https://github.com/TheAndrewStaker/mcp-midi-tools and you
   can read every line including the install scripts. To dismiss the
   warning, right-click the downloaded ZIP, choose Properties, tick
   Unblock, then extract.

Q: setup.cmd ran but Claude Desktop doesn't see the tools.
A: Make sure you fully quit Claude Desktop (system tray right-click
   then Quit) before reopening. Closing the window with X leaves it
   running in the background, and it only reads the config at
   startup.

Q: Will the AI delete my custom presets?
A: It can't, by default. Saves require explicit save-language from
   you, and the server refuses silent saves. See "WILL THIS WRECK MY
   PRESETS?" above and SAFETY-FOR-MUSICIANS.md on GitHub.

Q: What if the AI saves something I didn't want?
A: Fractal's factory presets are recoverable without a backup — call
   the tool "restore A01 to factory" (or any range) and the slot
   returns to factory bytes. AM4-Edit also exports full-flash .syx
   backups if you want extra insurance.

Q: Can I move this folder after install?
A: Yes, but you'll need to re-run setup.cmd from the new location so
   the config points at the right paths.
